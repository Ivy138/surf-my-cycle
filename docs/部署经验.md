## GitHub Pages 部署流程

### 方式一：GitHub API（推荐，已验证）

**只需要 Personal Access Token，无需浏览器登录**

1. 生成 Token
   - 地址: https://github.com/settings/tokens/new
   - 勾选 `repo` 权限
   - 生成后复制（格式: `ghp_xxx`）

2. 使用 API 启用 GitHub Pages
   ```bash
   curl -L -X POST "https://api.github.com/repos/{owner}/{repo}/pages" \
     -H "Authorization: Bearer {TOKEN}" \
     -H "Content-Type: application/json" \
     -d '{"build_type": "workflow", "source": {"branch": "main", "path": "/"}}'
   ```

3. 推送代码并访问
   ```bash
   git add index.html
   git commit -m "Deploy to GitHub Pages"
   git push -u origin main
   ```

### 方式二：手动开启
1. 访问: https://github.com/{owner}/{repo}/settings/pages
2. Source → Deploy from a branch → main → / (root) → Save
3. 等待 1-2 分钟

## 部署脚本

项目已有两个部署脚本：
- `deploy_github.py` - 基础版，使用 git 命令
- `deploy_api.py` - 使用 GitHub API，无需浏览器

## 注意事项

1. **HTML 文件必须命名为 `index.html`** - GitHub Pages 从根目录读取
2. **GitHub Pages 启用需要 1-2 分钟生效**
3. **Token 安全** - 不要将 token 提交到代码仓库，已在 `.gitignore` 中忽略
4. **每次推送自动部署** - GitHub Pages 默认启用 CI/CD

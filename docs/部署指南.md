## 🚀 部署步骤

### 步骤1：执行数据库迁移

1. 登录 Supabase Dashboard
2. 进入 SQL Editor
3. 打开 `supabase/migrations/001_memory_system.sql`
4. 执行SQL脚本
5. 确认以下表已创建：
   - `user_memories` - 用户长期记忆
   - `cycle_insights` - 周期洞察
   - `conversation_learnings` - 对话学习
   - `memory_extraction_logs` - 提取日志

---

### 步骤2：配置Vercel环境变量

在 Vercel 项目设置中添加以下环境变量：

```
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
MINIMAX_API_KEY=your-minimax-api-key
```

**获取方式**：
- `SUPABASE_URL`: Supabase项目设置 → API → URL
- `SUPABASE_SERVICE_ROLE_KEY`: Supabase项目设置 → API → service_role key
- `MINIMAX_API_KEY`: MiniMax控制台 → API Keys

---

### 步骤3：部署API

```bash
# 1. 确保文件已复制到正确位置
cp api/memory-extract.js /path/to/project/api/
cp api/memory-retrieve.js /path/to/project/api/

# 2. 部署到Vercel
vercel --prod

# 3. 验证API部署成功
curl https://your-app.vercel.app/api/memory-retrieve \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 步骤4：前端集成

#### 4.1 引入记忆集成脚本

在 `index.html` 中，在 `app.js` 之前引入：

```html
<script src="js/memory-integration.js"></script>
<script src="js/app.js"></script>
```

#### 4.2 修改 `js/app.js`

**修改1：替换buildAISystemPrompt调用**

找到这行：
```javascript
const systemPrompt = buildAISystemPrompt({
```

替换为：
```javascript
const systemPrompt = await buildAISystemPromptWithMemory({
```

**修改2：增强记忆学习调用**

找到这行：
```javascript
await learnFromConversation(msg, reply, cycleDay);
```

替换为：
```javascript
// 原有的学习逻辑
await learnFromConversation(msg, reply, cycleDay);

// 新增：Hermes风格记忆提取
await extractAndSaveMemories(msg, reply, {
  cycleDay,
  currentRecord: await getRecordWithDrafts(currentRecDate),
  recentRecords: await getRecentRecords(7)
});
```

**修改3：清除记忆缓存（当用户修改设置时）**

在 `saveUserPanel()` 函数中添加：

```javascript
// 保存成功后清除记忆缓存
memoryManager.clearCache();
```

## 🔧 配置选项

### 环境变量参考

| 变量名 | 必需 | 默认值 | 说明 |
|-------|------|--------|------|
| `SUPABASE_URL` | 是 | - | Supabase项目URL |
| `SUPABASE_SERVICE_ROLE_KEY` | 是 | - | Supabase服务角色密钥 |
| `MINIMAX_API_KEY` | 是 | - | MiniMax API密钥 |
| `MEMORY_CACHE_TTL` | 否 | 300000 | 记忆缓存TTL（毫秒） |
| `MIN_MEMORY_CONFIDENCE` | 否 | 0.6 | 最低记忆置信度 |

### 前端配置

在 `js/memory-integration.js` 顶部修改：

```javascript
const MEMORY_CONFIG = {
  // 缓存时间（毫秒）
  cacheTTL: 5 * 60 * 1000,
  
  // 默认检索记忆数量
  defaultLimit: 10,
  
  // 最低置信度
  minConfidence: 0.6,
  
  // 是否启用调试日志
  debug: true
};
```

## 🐛 故障排查

### 问题1：API返回401

**原因**：JWT Token无效或过期
**解决**：
1. 检查用户是否已登录
2. 确认token未过期
3. 重新登录获取新token

### 问题2：记忆提取失败

**原因**：MiniMax API错误或网络问题
**解决**：
1. 检查 `MINIMAX_API_KEY` 是否正确
2. 查看Vercel函数日志
3. 确认MiniMax账户有余额

### 问题3：记忆未注入到AI对话

**原因**：`buildAISystemPromptWithMemory` 未被调用
**解决**：
1. 确认已替换原有函数调用
2. 检查浏览器控制台是否有 `已加载记忆` 日志
3. 确认 `memory-integration.js` 已正确加载

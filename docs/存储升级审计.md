# 存储升级审计报告

**日期**：2026-03-24
**版本**：V4
**审计人**：Cascade

---

## 变更概述

将数据存储从 `localStorage` 升级为 `IndexedDB`，解决容量和性能问题。

---

## 技术变更

### 1. 新增依赖
- 引入 `idb@8.0.0` 库（CDN）

### 2. 数据库设计

| Store | 用途 | KeyPath |
|-------|------|---------|
| records | 日记录 | date |
| conversations | AI对话 | date |
| config | 配置 | key |
| memory | 记忆 | key |

### 3. 新增函数

```javascript
// 核心
initDB()           // 初始化数据库
getData()          // 获取所有记录（async）
setData()          // 保存记录（async）
getConfig()        // 获取配置（async）
setConfig()        // 保存配置（async）
getRecord()        // 获取单条记录（async）
saveRecordDB()     // 保存单条记录（async）

// AI对话
getAIConversation()    // 获取AI对话（async）
saveAIConversation()   // 保存AI对话（async）

// 记忆
getMemory()       // 获取记忆（async）
setMemory()       // 保存记忆（async）

// 导入导出
exportData()      // 导出JSON（async）
importData()      // 导入JSON（async）

// 兼容
getDataLegacy()   // 读取旧localStorage
getConfigLegacy() // 读取旧配置
getRecordLegacy() // 读取旧记录
```

### 4. 函数签名变更（async）

以下函数改为 async：
- `renderCalendar`
- `loadRecordPanel`
- `saveRecord`
- `getCycleDay`
- `renderAnalysis`
- `openSettings`
- `saveSettings`
- `sendAI`
- `getRecentAIMemories`

---

## 数据迁移

保留旧版 localStorage 兼容函数（getDataLegacy等），可用于数据迁移。

首次启动时如需迁移旧数据，可调用：
```javascript
// 迁移示例
const oldData = getDataLegacy();
await setData(oldData);
```

---

## 导出/导入功能

- **导出**：`exportData()` - 下载JSON文件
- **导入**：`importData(jsonStr)` - 从JSON恢复

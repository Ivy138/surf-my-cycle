# 配置Vercel环境变量 - 3分钟完成

## 需要配置的3个变量

### 1. SUPABASE_URL
**获取位置：** https://supabase.com/dashboard
- 登录 Supabase
- 选择你的项目
- 左侧菜单 Settings → API
- 复制 **"Project URL"** 的值

**格式：** `https://xxxxxxxxxxxxxxxxxxxx.supabase.co`

---

### 2. SUPABASE_SERVICE_ROLE_KEY
**同一个页面：**
- 在 Settings → API 页面
- 找到 **"service_role"** 对应的 key
- 点击 Copy

⚠️ **注意：** 这是高权限密钥，不要泄露给他人

---

### 3. MINIMAX_API_KEY
**获取位置：** https://www.minimaxi.com/platform
- 登录 MiniMax 开放平台
- 进入控制台
- 找到 API Key 管理
- 复制你的 API Key

---

## 配置步骤（2分钟）

### 方法A：Vercel Dashboard（推荐）

1. 访问 https://vercel.com/dashboard
2. 找到 `surf-my-cycle` 项目，点击进入
3. 顶部菜单点击 **"Settings"**
4. 左侧菜单点击 **"Environment Variables"**
5. 逐个添加以下变量：

| Name | Value |
|------|-------|
| `SUPABASE_URL` | 上面复制的URL |
| `SUPABASE_SERVICE_ROLE_KEY` | 上面复制的service_role key |
| `MINIMAX_API_KEY` | 上面复制的MiniMax key |

6. 点击 **"Save"**
7. Vercel会自动重新部署（约1-2分钟）

---

### 方法B：使用.env文件（如果本地开发）

1. 复制 `.env.example` 为 `.env.local`
2. 填入3个真实值
3. 运行 `npx vercel env pull` 同步到云端

---

## 验证配置成功

配置完成后，打开浏览器访问：
```
https://surf-my-cycle.vercel.app
```

1. 登录账号
2. 进入AI对话
3. 说："我不喜欢黄体期被提醒吃太多"
4. AI应该回复并记住这个偏好

---

## 遇到问题？

如果配置后API仍然报错：
1. 检查3个值是否复制完整（特别是key不要有截断）
2. 确认Vercel显示 "Production Deployment" 已完成
3. 打开浏览器控制台查看具体错误
